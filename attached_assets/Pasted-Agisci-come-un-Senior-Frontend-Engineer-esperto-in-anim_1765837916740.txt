Agisci come un Senior Frontend Engineer esperto in animazioni CSS e UI fluide.

Ho un problema con il componente "InfiniteCarousel" attuale. Come mostrato nella registrazione dello schermo, l'animazione di scorrimento orizzontale presenta un "glitch" o "scatto" visibile alla fine del ciclo. Invece di scorrere all'infinito in modo impercettibile, si vede chiaramente il momento in cui l'animazione si resetta o riparte.

Il carosello contiene card come "Sedi", "News", "Contatti", "Attività".

Per favore, riscrivi o correggi la logica del carosello seguendo rigorosamente queste specifiche tecniche per garantire un loop perfetto:

1.  **Strategia di Duplicazione (Cloning):**
    Il contenuto scorrevole deve contenere due set identici di elementi (Set A + Set B identico).
    Struttura ideale: `Container > [Wrapper Scorrevole] > [Lista Originale] + [Lista Clonata]`.

2.  **Calcolo dell'Animazione (Keyframes):**
    L'animazione CSS deve spostare il wrapper da `translateX(0)` a `translateX(-50%)` (se il wrapper contiene il doppio degli elementi).
    *IMPORTANTE:* Il punto finale `-50%` deve corrispondere ESATTAMENTE all'inizio visivo del primo elemento della lista clonata. Se c'è un pixel di differenza o se il `gap` tra gli elementi non è incluso nel calcolo, si vedrà lo scatto.

3.  **Gestione del Gap/Margine:**
    Assicurati che lo spazio (gap) tra l'ultimo elemento della Lista Originale e il primo elemento della Lista Clonata sia identico allo spazio tra gli altri elementi. Spesso l'errore è qui: manca un margine alla fine della prima lista.

4.  **Fluidità:**
    Usa `animation-timing-function: linear;` per evitare accelerazioni o decelerazioni che rendono il loop meno naturale.
    Usa `will-change: transform;` per ottimizzare le performance di rendering.

5.  **Responsive:**
    Assicurati che la logica funzioni anche se la larghezza dello schermo cambia (percentuali o flexbox sono preferibili a larghezze fisse in pixel hardcoded).

**Output richiesto:**
Forniscimi il codice CSS e HTML (o JSX) corretto. Se serve JavaScript per calcolare la larghezza dinamica, includi anche quello, ma preferisco una soluzione CSS-pure se possibile. Spiegami brevemente dove era l'errore.
