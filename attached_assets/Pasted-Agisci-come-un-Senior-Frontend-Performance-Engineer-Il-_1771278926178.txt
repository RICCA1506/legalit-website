1. CONFIGURAZIONE BUILD & SERVE (Cruciale):
Attualmente il progetto potrebbe usare un server di sviluppo (es. `vite` o `npm run dev`). Questo è lento per la produzione.
- Modifica il file `package.json` e il file `.replit`.
- Imposta il comando `run` in modo che esegua prima la build (es. `npm run build`) e poi serva la cartella statica generata (es. `dist` o `build`) usando un pacchetto performante come `serve` o un server Node/Express ottimizzato per file statici con compressione Gzip/Brotli attivata.
- Il comando finale nel .replit deve assomigliare a: `run = "npm run build && npx serve -s dist -l 3000"` (adatta la cartella in base al framework).

2. ASSET STRATEGY & PRELOADING (Stile "LandonNorris"):
- Scansiona il file `index.html` e i componenti principali.
- Aggiungi `<link rel="preload">` nell'head per le risorse critiche (font principali, immagine hero, logo) per alzare il punteggio LCP (Largest Contentful Paint).
- Applica `loading="lazy"` e `decoding="async"` a tutte le immagini che non sono nella viewport iniziale (below the fold).
- Se rilevi importazioni di librerie pesanti (come Three.js, Framer Motion, o grandi librerie di UI), implementa il Code Splitting usando `React.lazy` / `Suspense` (o equivalente per il framework in uso) per non bloccare il caricamento iniziale.

3. GESTIONE COLD START (Mascheramento):
- Poiché Replit va in sleep, crea/ottimizza un "Skeleton Loader" o un preloader CSS ultraleggero inline direttamente nell'`index.html` (prima che carichi il bundle JS).

4. VITE/WEBPACK CONFIG (Se applicabile):
- Controlla `vite.config.js` (o webpack). Assicurati che la minificazione sia attiva (terser/esbuild) e abilita `splitVendorChunk` per separare le librerie esterne dal codice sorgente, permettendo al browser di metterle in cache.
